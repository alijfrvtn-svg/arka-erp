# راهنمای اجرای سامانه Arka ERP

این سند دقیقاً می‌گوید هر بار برای بالا آوردن یا خاموش‌کردن سیستم چه کنید.

پیش‌نیاز: فقط **Docker Desktop** باید نصب و در حال اجرا باشد. (نیازی به نصب Node/Postgres روی سیستم نیست.)

آدرس مسیر پروژه: `F:\Amirhossein files\arka-erp`

---

## ۱) نخستین اجرا (یا بعد از تغییر کد)

یک بار PowerShell را باز کنید و این را اجرا کنید:

```powershell
cd "F:\Amirhossein files\arka-erp"
docker compose up -d --build
```

- بار اول ۳ تا ۵ دقیقه طول می‌کشد (ساخت ایمیج‌ها + راه‌اندازی دیتابیس).
- دیتابیس فقط **یک‌بار** در نخستین اجرا مقداردهی می‌شود (سرفصل حساب‌ها، دوره‌های مالی، داده‌های نمونه، کاربر مدیرعامل).

سپس مرورگر را باز کنید:

> **http://localhost:8080**

---

## ۲) اجرای روزمره (کد تغییر نکرده)

```powershell
cd "F:\Amirhossein files\arka-erp"
docker compose up -d
```

بدون `--build` سریع بالا می‌آید (چند ثانیه). سپس **http://localhost:8080**.

> اگر Docker Desktop بسته بوده، اول آن را باز کنید و ۳۰ ثانیه صبر کنید تا موتورش بالا بیاید، بعد دستور بالا را بزنید.

---

## ۳) توقف سیستم

```powershell
cd "F:\Amirhossein files\arka-erp"
docker compose stop
```

- `stop` فقط کانتینرها را خاموش می‌کند؛ **همهٔ داده‌ها حفظ می‌شوند**. دفعهٔ بعد با `docker compose up -d` همان داده‌ها برمی‌گردند.

---

## ۴) بررسی وضعیت و لاگ‌ها

```powershell
docker compose ps                 # وضعیت کانتینرها (باید healthy باشند)
docker compose logs -f backend    # لاگ زندهٔ بک‌اند
docker compose logs -f frontend   # لاگ زندهٔ فرانت‌اند
```

سلامت سیستم از داخل خود برنامه هم قابل مشاهده است: منوی **سامانه ← سلامت سیستم** (با نقش مدیرعامل).

---

## ۵) حساب‌های ورود

| نقش | ایمیل | گذرواژه | کاربرد |
|-----|-------|---------|--------|
| مدیرعامل (CEO) | `ali.jafari@arka.studio` | `Arka@2026!` | مشاهدهٔ همه‌چیز، مدیریت کاربران، برگشت سند |
| حسابدار | `accountant@arka.studio` | `Arka@2026!` | ثبت و قطعی‌سازی اسناد مالی |

> **توجه:** ثبت سند مالی فقط با نقش **حسابدار** ممکن است (مدیرعامل طبق تفکیک وظایف فقط مشاهده/برگشت دارد). کاربرهای جدید و نقش‌هایشان را از منوی **سامانه ← مدیریت کاربران** بسازید.

> **دربارهٔ MFA:** اگر روی حسابی احراز دومرحله‌ای فعال کرده‌اید، هنگام ورود کد ۶ رقمی خواسته می‌شود. اگر به کد دسترسی ندارید، با حساب مدیرعامل وارد شوید → **مدیریت کاربران** → کنار آن کاربر دکمهٔ **بازنشانی** MFA را بزنید.

---

## ۶) کارهای نگهداری (اختیاری)

**شروع کاملاً از صفر (پاک‌کردن همهٔ داده‌ها و مقداردهی مجدد دیتابیس):**
```powershell
cd "F:\Amirhossein files\arka-erp"
docker compose down -v          # -v یعنی حذف volumeها (همهٔ داده پاک می‌شود!)
docker compose up -d --build
```
> این کار همهٔ اسناد، کاربران و پروژه‌های ثبت‌شده را حذف و سیستم را به حالت نصب اولیه برمی‌گرداند.

**فقط توقف بدون حذف کانتینرها:** `docker compose stop`
**حذف کانتینرها ولی حفظ داده:** `docker compose down` (بدون `-v`)

## پیش از استفادهٔ واقعی در کسب‌وکار (مهم)

این پیکربندی «توسعه/دمو» است. قبل از استفادهٔ واقعی، در فایل `.env` این‌ها را تغییر دهید:
- `JWT_ACCESS_SECRET`, `JWT_STEPUP_SECRET` → مقادیر تصادفی بلند
- `ENVELOPE_MASTER_KEY_HEX` → ۳۲ بایت تصادفی (۶۴ کاراکتر hex)
- `DB_PASSWORD` و گذرواژهٔ کاربران پیش‌فرض
- فعال‌سازی HTTPS/TLS روی nginx و راه‌اندازی پشتیبان‌گیری خودکار
